# Interview Talking Points

## 30-Second Introduction

I am a computer science student focused on AI, Python automation, and practical web tools. My strongest current project is a rental research report generator that filters listing data, ranks housing options, and generates PDF reports with budgets and checklists. I am looking for remote internships, freelance projects, or part-time developer roles where I can build useful tools, document them clearly, and learn from production feedback.

## Featured Project: Rental Research Report Generator

### What It Does

It takes rental listing data, removes unsuitable options, ranks the remaining homes, and generates a decision-ready PDF report.

### Why It Matters

It solves a real local problem. It is not a clone or a tutorial. It shows that I can turn messy real-world information into structured output that someone can actually use.

### Technical Skills Shown

- Python scripting
- Data cleaning
- Filtering and scoring logic
- JSON input/output
- PDF generation
- CLI design
- Unit testing
- Documentation

### Hard Part

The hard part was handling imperfect public data. Many fields are hidden, missing, or unreliable, so the tool had to be honest about what it knows and what must be verified manually.

### What I Learned

I learned that useful software is not only about code. It also needs clear outputs, repeatable demos, honest limitations, and tests around the logic that matters.

## Common Questions

### Why should we hire you for a remote internship?

I can take a practical problem, break it into small parts, build a working version, document it, and improve it with feedback. I am comfortable using Python, React, GitHub, and AI-assisted workflows, and I communicate progress clearly.

### What kind of work are you strongest at?

I am strongest at automation, reporting, dashboards, data cleanup, and small tools that reduce manual work.

### What do you still need to improve?

I want to improve backend API design, production deployment, testing depth, and working with real team codebases.

### How do you use AI tools?

I use AI tools to speed up research, scaffolding, debugging, and refactoring, but I still verify code by running tests, checking outputs, and understanding the final behavior.

### What project would you build next?

I would build an AI document assistant, a React dashboard with filters and CSV export, or a FastAPI backend that turns uploaded JSON into generated reports.

## Questions To Ask Employers

- What would the first two weeks of work look like?
- How do you review code from interns or junior developers?
- Are tasks usually frontend, backend, automation, or mixed?
- What does successful remote communication look like on your team?
- Are there existing docs or tests I should read first?
